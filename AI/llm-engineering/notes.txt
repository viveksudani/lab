# Day 2 Notes ###############################################

- Understand API call for chat completions
- Uderstand OpenAI class is just API call wrapper and also supports diffrent models via base_url & auth_key
- OpenAI api class can be used to interact with locally running models via ollama as well:

from openai import OpenAI
OLLAMA_BASE_URL = "http://localhost:11434/v1"

ollama = OpenAI(base_url=OLLAMA_BASE_URL, api_key='ollama')
response = ollama.chat.completions.create(model="phi3", messages=[{"role": "user", "content": "Tell me a fun fact"}])

response.choices[0].message.content

# Day 4 Notes ###############################################
GPT - Generative Pre-trained Transformer

Attention is all you need! - paper -- 2017

Parameters
GPT1 - 117M
GPT2 - 1.5B
GPT3 - 175B
GPT4 - 1.76T
More parameters -- More time to train

Tokens - https://platform.openai.com/tokenizer

1. Every call to an LLM is stateless
2. We pass in the entire conversation so far in the input prompt, every time
3. This gives the illusion that the LLM has memory - it apparently keeps the context of the conversation
4. But this is a trick; it's a by-product of providing the entire conversation, every time
5. An LLM just predicts the most likely next tokens in the sequence; if that sequence contains "My name is Ed" and later "What's my name?" then it will predict.. Ed!

- LLMs predict token by token.Predicts one token, then entire sentence is again sent to LLM as input to predict next token.
